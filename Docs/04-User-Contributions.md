# 04 — User Contributions（使用者投稿）

> 一般使用者可以提供資料的功能。前端 UX + 投稿流程 + 限制與防濫用。

---

## 1. 設計目標

讓社群可以「挑錯字、補漏字、補圖、補影片、新增遺漏遊戲」，但不會被惡意投稿淹沒、也不會誤改正資料。核心是：

1. **投稿低門檻**：點一個按鈕就能投，欄位最少
2. **即時可見**：投稿 → 進待審佇列，在 detail page 顯示「2 筆待審變更」
3. **審核透明**：使用者看得到自己投稿的狀態 + reject 原因
4. **可逆**：沒有資料因投稿就直接被覆蓋；都要管理員核可

### 2.3 入口 3：主目錄的「➕ 新增遺漏遊戲」

當使用者搜不到某款遊戲，搜尋空結果頁顯示：

```
找不到「Foo Bar」？

```

點下去開新增遊戲表單（最少需填：主機 / 標題 / 廠商 / 年份）。

## 3. 投稿類型

| Type | 範例 | 必填欄位 |
|---|---|---|
| `add_game` | 新增遊戲整筆 | console, title_en, year |
| `edit_field` | 改某欄位（標題、廠商、類型…） | game_id, target_field, new_value |
| `add_translation` | 加某語言標題 / 簡介 | game_id, language, kind, text |
| `add_image` | 上傳封面 / 截圖 | game_id, kind, file |
| `add_video` | 加 YouTube link | game_id, youtube_url |
| `report_error` | 純舉報，不附建議 | game_id, comment |

每個類型對應後端 `contributions.type` ENUM。

## 5. 防濫用機制

### 5.1 限額
| 限制 | 值 |
|---|---|
| 註冊驗證 | email verify 必填，未驗證不能投稿 |
| 每日投稿上限 | 20 筆 / user |
| 圖片上傳 | 5 MB max，jpg/png/webp，max 50/天 |
| hCaptcha | 第一次投稿、被駁過後加 captcha |

### 5.2 聲望系統（`users.reputation`）
- 投稿通過：**+1 分**
- 投稿被駁回：**-2 分**
- 連續 3 筆通過：**+5 bonus**
- 持有 reputation ≥ 50：可選 `contributor` role（投稿不需要 captcha、提示給 mod 為「老 user」）
- reputation < -10：自動鎖定投稿 7 天

### 5.3 黑名單
- 同一 IP 24 小時內投稿超過 50 次 → 加 IP 觀察名單，投稿暫停
- 重複投稿同一個錯誤內容（payload hash 重複 ≥ 3 次）→ 自動標記 duplicate

## 6. 流程圖（happy path）

```
使用者點 ✎ 修改
   ↓
填表單 → 前端 validate
   ↓
POST /api/v1/contributions
   ↓
後端：
  1. 驗 token / role
  2. 驗 quota（不超過每日 20）
  3. 寫入 contributions（status=pending）
  4. 觸發 ProcessContribution job
   ↓
Job：
  - 圖片：跑 image processing（resize / upload R2）
  - 通知 moderator (Slack webhook 或站內 inbox)
   ↓
等待管理員審核 → 看 05-Admin-Moderation.md
   ↓
Approved：
  - 套用變更到 games / titles / descriptions
  - +1 reputation
  - 通知投稿者（email + 站內 toast）
   ↓
Rejected：
  - -2 reputation
  - 通知投稿者 + rejection_reason
```

## 7. UI 細節

### 7.1 detail page 顯示「待審」

當某遊戲有 pending contribution 時，detail page 該欄位旁顯示一個「⏳ 1 筆待審」小 chip：

```
中文標題：銀河戰士融合  ⏳1     ✎
                       ↓ hover
                       「Taffy 提議改成『銀河戰士：融合』
                         尚未審核」
```

不公開顯示投稿者名字直到通過。

### 7.2 投稿成功 toast

```
✓ 投稿已送出，編號 #999
  審核通常 1-3 天，通過後會 email 通知。
  [ 查看我的投稿 ]
```

### 7.3 駁回時的解釋（透明）

```
❌ 你的投稿被駁回了
   遊戲：1941 · cover
   駁回原因：附圖解析度過低
   管理員留言：請使用 ≥ 320×480 的圖
   [ 重新投稿 ]   [ 查看詳情 ]
```

## 8. 註冊 / 登入

### 8.1 流程

```
方式 A：email + password
  ┌────────────────────┐
  │ Email: ___________ │
  │ 密碼: ___________ │
  │ [ 註冊 ]          │
  └────────────────────┘
         ↓
  寄驗證信
         ↓
  點信中連結 → email_verified_at 寫入
         ↓
  可投稿

方式 B：Google OAuth（建議首推）
  ┌────────────────────┐
  │ [ Google 登入 ]   │
  └────────────────────┘
         ↓
  跳轉 Google → 同意
         ↓
  Callback → 自動建 user（email_verified_at 預先設）
         ↓
  可立即投稿
```

### 8.2 不需要的東西
- 大頭貼上傳：用 OAuth 帶來的 avatar 即可
- 暱稱：從 OAuth display_name 或 email 預設帶
- 簡介 / 個人頁：不做（不是社群網站）

## 9. 邊界 / 開放問題

1. **匿名投稿是否允許？**
   - **建議：不允許**。註冊門檻雖然會少投稿量，但能大幅減少 spam 與濫用。
   - 折衷：允許「匿名舉報錯誤」（type=report_error），但不能 add/edit。

2. **投稿者要不要公開掛名？**
   - 可選：投稿者 setting 裡有「投稿通過後公開掛名 ✓/✗」
   - 公開時，detail page footer 顯示「最近 10 筆貢獻者：A, B, C…」

3. **同一遊戲 multiple pending edits 怎麼辦？**
   - 顯示「2 筆待審變更」chip，moderator 可以 batch approve / reject
   - 同一欄位多人投稿不同值 → moderator 挑一筆 approve，其他自動 dup

4. **投稿後要不要可以即時 preview？**
   - **建議**：上傳圖片時前端即時 preview（不打 server）
   - 文字欄位投稿不 preview，因為要等通過

5. **多語言投稿介面？**
   - v1：投稿表單只中文
   - v2：依 user 偏好支援 EN / JP
